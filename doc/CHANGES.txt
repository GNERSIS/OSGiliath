===============================================================================
     OSGiliath — OpenSceneGraph Modernized for OpenGL 4.6 Core Profile
===============================================================================

  This fork removes ALL fixed-function and deprecated OpenGL calls from
  OpenSceneGraph, targeting a clean GL 4.6 Core Profile with no backward
  compatibility.  ~33,000 lines of legacy code deleted across 550+ files
  in the fixed-function removal alone; thousands more changed across
  examples, shaders, and the build system.

===============================================================================
  LAYER 1 — Build System & Platform
===============================================================================

  CMake .............. Targets GL 4.6 Core Profile exclusively
  Platform code ...... Apple/iOS/Android/GLES targets removed
  Extension loading .. GLEW integrated (replaces per-platform loaders)
  Feature flags ...... OSG_GL_FIXED_FUNCTION_AVAILABLE     -> deleted
                       OSG_GL_MATRICES_AVAILABLE           -> deleted
                       OSG_GL_VERTEX_ARRAY_FUNCS_AVAILABLE -> deleted
                       OSG_GL1/GL2/GLES1/GLES2 features   -> deleted
                       Only OSG_GL3_FEATURES remains (always ON)
  GL.in template ..... Stripped to GL3-only defines
  DSA ................ Direct State Access conversion begun
                       (textures, buffers, framebuffers)

===============================================================================
  LAYER 2 — Examples & Shaders
===============================================================================

  ~120 shader files .. Updated from legacy GLSL to 460 core
  Lighting ........... Blinn-Phong via uniform structs (osg_Light, etc.)
  Materials .......... Material properties via uniform structs
  Fog ................ Fog parameters via uniform structs
  Geometry ........... GL_QUADS -> GL_TRIANGLES throughout
  VBOs ............... Enforced everywhere (no display lists)
  Fixed-function ..... Replaced with GLSL shaders in all examples

===============================================================================
  LAYER 3b — Runtime Regression Fixes (Core Profile)
===============================================================================

  After the large-scale code deletion and shader modernization, many
  examples failed to render at runtime. These fixes address the runtime
  interactions between the shader pipeline, vertex array state, and
  Core Profile GL constraints.

  PHASE 1 — VBO/VAO Infrastructure (72/75 regressions fixed):
  -----------------------------------------------------------------------
  Geometry.cpp .............. Constructor defaults _useVertexBufferObjects=true;
                              addVertexBufferObjectIfRequired() assigns VBOs to
                              ALL arrays (not just BIND_PER_VERTEX)
  TextBase.cpp .............. compileGLObjects() no longer calls drawImpl()
                              (crashed without shader); compiles VBO/EBO data
                              and sets up VAO vertex attribute bindings separately
  VertexArrayState .......... After compileBuffer() (DSA, no binding),
                              bindBuffer() now called to actually bind buffer
  VAS stale cache ........... After compileGLObjects sets up VAO,
                              resetBufferObjectPointers() called so rendering
                              re-binds buffers (GL_ARRAY_BUFFER not in VAO state)

  PHASE 2 — Texture Rendering:
  -----------------------------------------------------------------------
  Texture::getModeUsage ..... Restored unconditionally (was behind #ifdef) so
                              setTextureAttributeAndModes() populates
                              _textureModeMapList, enabling texture shader defines
  applyModeOnTexUnit ........ Modified to only update last_applied_value without
                              calling glEnable(GL_TEXTURE_2D) (invalid in Core)
  TEXTURE_ENV_FUNCTION ...... Added #define TEXTURE_ENV_FUNCTIONi texenv_MODULATE
                              to texture mode define strings in State.cpp

  PHASE 3 — Vertex Attribute Aliasing & Mode Filtering:
  -----------------------------------------------------------------------
  Vertex attribute aliasing .. Changed resetVertexAttributeAlias(true) to
                              resetVertexAttributeAlias(false) (non-compact).
                              Shader pipeline hardcodes layout(location) matching
                              OpenGL conventional indices (Vertex=0, Normal=2,
                              Color=3, TexCoord0=8); compact aliasing assigned
                              incompatible slots causing wrong texture coords
  applyMode Core Profile .... Added isModeApplicable() whitelist; applyMode()
                              only calls glEnable/glDisable for valid Core Profile
                              modes (GL_BLEND, GL_DEPTH_TEST, GL_CULL_FACE, etc.).
                              Fixed-function modes still tracked for shader defines
  Fixed-function re-enable .. Removed setUseStateAttributeFixedFunction(true) from
                              initializeExtensionProcs() — was overriding the
                              constructor's false setting

  PHASE 4 — Lighting & Material Uniforms:
  -----------------------------------------------------------------------
  Default white vertex color . In Core Profile, disabled vertex attributes
                              default to (0,0,0,1) — black. Added
                              state.Color(1,1,1,1) in Geometry::draw when
                              _colorArray is null, matching legacy GL initial
                              color state
  Light as StateAttribute ... Changed setAssociatedModes() to
                              setAttributeAndModes() in LightSource and
                              SceneView so Light::apply() is called during
                              state application, delivering actual light
                              parameters to shader uniforms
  External shader files ..... data/shaders/shaderpipeline.vert/frag and
                              shadergen.vert/frag replaced with Core Profile
                              versions matching embedded shaders

  PHASE 5 — TexGen Sphere-Map:
  -----------------------------------------------------------------------
  osg::TexGen re-added ...... Minimal TexGen class (no GL calls, mode tracking
                              only). Serializer enables cow.osgt loading.
                              GL_TEXTURE_GEN_S/T/R/Q added to TextureGLModeSet,
                              _stringModeMap, glTable. Shader pipeline generates
                              sphere-map tex coords when GL_TEXTURE_GEN_S active.
                              ShaderGen path also updated with OSG_TEXGEN_SPHERE_MAP

  PHASE 6 — GL_COLOR_MATERIAL Emulation (initial):
  -----------------------------------------------------------------------
  Shader lighting modulation . Shader lighting computes
                              matDiffuse = osg_FrontMaterial.diffuse * osg_Color
                              When osg_Color is white (default), material is
                              unchanged; when vertex colors present, they
                              modulate material properties
  Zero-normal detection ..... Default normal set to (0,0,0) when no normal
                              array. Shader detects length < 0.001 and uses
                              NdotL=1.0 for full illumination (no-normals case)
  VAS default attributes .... disableColorArray() sets glVertexAttrib4f white,
                              disableNormalArray() sets (0,0,0,0). Ensures
                              correct defaults switching between geometries
  Drawable serializer compat . Read-only serializers for UseDisplayList/
                              SupportsDisplayList so old .osgt files load

  PHASE 7 — Texture Mode Define Timing:
  -----------------------------------------------------------------------
  applyModeOnTexUnit ........ Now sets _defineMap.changed=true when texture
                              mode changes, triggering Program re-application
                              with updated defines
  getDefineString ........... Changed from valueVec.back() to last_applied_value
                              for all 3 mode lookups (apply(StateSet*) path uses
                              applyModeListOnTexUnit which updates
                              last_applied_value without pushing to valueVec)
  TEXTURE_ENV_FUNCTION ...... Now generated dynamically in getDefineString()
                              with MODULATE default, overridable via setDefine()

  PHASE 8 — Point & Spot Light Support:
  -----------------------------------------------------------------------
  Point lights .............. Added position.w check for directional vs point.
                              Point lights use L = normalize(pos - eyeVertex)
                              with distance attenuation 1/(c + l*d + q*d²).
                              Half vector uses local viewer for point lights
  Spot lights ............... Spot cone cutoff: dot(-L, spotDir) >= cosCutoff
                              with pow(spotDot, exponent) falloff
  osgSSBO shaders ........... All 4 data shaders rewritten for Core Profile:
                              attribute→layout(location), gl_PositionIn→gl_in,
                              texture2D→texture, gl_FragColor→out vec4
  osgSSBO + osgstereoimage .. GL_QUAD_STRIP → GL_TRIANGLE_STRIP

  PHASE 9 — SceneView Headlight, osglogo, osgParticle:
  -----------------------------------------------------------------------
  SceneView headlight ....... setAssociatedModes() → setAttributeAndModes()
                              in SceneView constructor and setLight() so
                              Light::apply() delivers params to shader uniforms
  osglogo earth texture ..... Custom shader updated: added osg_MultiTexCoord0,
                              sampler2D baseTexture, bool hasTexture uniforms
  osgParticle QUAD path ..... Restored non-shader QUAD path with billboard
                              geometry, tile UV coords (was forcing GL_POINTS)

  PHASE 10 — DSA Buffer Binding, GL_COLOR_MATERIAL, LightModel:
  -----------------------------------------------------------------------
  DIBO bindBuffer ........... bindDrawIndirectBufferObject() was missing
                              bindBuffer() after DSA compileBuffer(). Same fix
                              as VBO/EBO. Fixes osgsimpleMDI rendering
  PBO bindBuffer ............ Same DSA pattern fix for pixel buffer objects
  GL_COLOR_MATERIAL ......... Added osg_ColorMaterial float uniform (0.0/1.0).
                              Geometry sets 1.0 when color array present.
                              Shader uses mix(material, vertexColor, flag) to
                              choose between material properties and vertex
                              color — proper REPLACE semantics vs multiply
  LightModel ambient ........ Added osg_LightModel_ambient vec4 uniform,
                              default (0.1,0.1,0.1,1.0) matching osg36
                              SceneView LightModel. Replaces hardcoded 0.2

  DELETED CLASSES (no core profile equivalent):
  -----------------------------------------------------------------------
  osg::AlphaFunc .............. -> shader discard
  osg::ShadeModel ............. -> always smooth; use flat qualifier
  osg::ColorMatrix ............ -> not in core profile
  osg::PointSprite ............ -> gl_PointCoord always available
  osg::TexEnv ................. -> shader texture sampling
  osg::TexEnvCombine .......... -> shader texture sampling
  osg::TexEnvFilter ........... -> shader texture sampling
  osg::TexGen ................. -> shader uniforms
  osg::TexGenNode ............. -> shader uniforms
  osg::FragmentProgram ........ -> osg::Program / osg::Shader (GLSL)
  osg::VertexProgram .......... -> osg::Program / osg::Shader (GLSL)
  osgVolume::FixedFunctionTechnique -> RayTracedTechnique

  KEPT AS DATA STORAGE (GL calls removed from apply()):
  -----------------------------------------------------------------------
  osg::Light .................. ambient/diffuse/specular as uniform source
  osg::Material ............... material properties as uniform source
  osg::LightModel ............. data storage only
  osg::Fog .................... fog parameters as uniform source
  osg::ClipPlane .............. REMOVED (no-op in core profile; use gl_ClipDistance in shaders)
  osg::ClipNode ............... REMOVED (depended on ClipPlane; non-functional in core profile)
  osg::TexMat ................. texture matrix as uniform
  osg::LogicOp ................ glLogicOp is core — kept as-is
  osg::Point .................. glPointSize is core — kept as-is

  DISPLAY LISTS -> DELETED:
  -----------------------------------------------------------------------
  DisplayListManager .......... deleted from Drawable
  glGenLists/glCallList ....... deleted
  COMPILE_DISPLAY_LISTS ....... renamed to COMPILE_VERTEX_BUFFER_OBJECTS
  RELEASE_DISPLAY_LISTS ....... renamed to RELEASE_VERTEX_BUFFER_OBJECTS
  SWITCH_ON/OFF_DISPLAY_LISTS . deleted (no-ops)
  DatabasePager USE_DISPLAY_LISTS -> deleted from DrawablePolicy

  CLIENT STATE ARRAYS -> DELETED:
  -----------------------------------------------------------------------
  glEnableClientState ......... deleted
  glVertexPointer ............. deleted
  glNormalPointer ............. deleted
  glColorPointer .............. deleted
  glTexCoordPointer ........... deleted
  glSecondaryColorPointer ..... deleted
  glFogCoordPointer ........... deleted
  glInterleavedArrays ......... deleted
  Only glVertexAttribPointer / VAO path remains

  MATRIX STACK -> DELETED:
  -----------------------------------------------------------------------
  glMatrixMode ................ deleted
  glPushMatrix / glPopMatrix .. deleted
  glLoadMatrix / glLoadIdentity deleted
  Replaced with State::applyModelViewMatrix() and uniforms

  ATTRIBUTE STACK -> DELETED:
  -----------------------------------------------------------------------
  glPushAttrib / glPopAttrib .. replaced with explicit state set/restore

  TEXTURE FORMATS:
  -----------------------------------------------------------------------
  GL_LUMINANCE ................ -> GL_RED  + swizzle {R,R,R,1}
  GL_LUMINANCE_ALPHA .......... -> GL_RG   + swizzle {R,R,R,G}
  GL_INTENSITY ................ -> GL_RED  + swizzle {R,R,R,1}
  GL_ALPHA (format) ........... -> GL_RED
  All sized variants .......... -> GL_R8/R16/R16F/R32F/RG8/RG16 etc.
  Compressed variants ......... -> GL_COMPRESSED_RED_RGTC1 etc.
  Swizzle applied in Texture::applyTexParameters() — zero runtime cost

  MIPMAP GENERATION:
  -----------------------------------------------------------------------
  GL_GENERATE_MIPMAP param .... -> glGenerateMipmap() after upload

  SMOOTHING HINTS -> DELETED:
  -----------------------------------------------------------------------
  GL_POINT_SMOOTH ............. -> use multisampling
  GL_LINE_SMOOTH .............. -> use multisampling
  GL_POLYGON_SMOOTH ........... -> use multisampling

  ARB ASSEMBLY PROGRAMS -> DELETED:
  -----------------------------------------------------------------------
  osg::FragmentProgram ........ deleted (header, source, serializers)
  osg::VertexProgram .......... deleted (header, source, serializers)
  ARB program extensions ...... removed from GLExtensions
  osgFX techniques ............ ARB program paths removed, GLSL kept
  osgvertexprogram example .... deleted

  MODULE-LEVEL CHANGES:
  -----------------------------------------------------------------------
  osgShadow ................... TexGen -> shader uniform matrices
  osgSim ...................... OverlayNode TexGenNode removed
                                LightPoint drawables modernized
  osgVolume ................... FixedFunctionTechnique deleted
  osgFX ....................... ARB programs removed, GLSL techniques kept
  osgText ..................... TexEnv removed, shader handles text color
  osgParticle ................. Matrix/attrib stack -> uniforms + state

  IFDEF SCAFFOLDING -> DELETED:
  -----------------------------------------------------------------------
  All #ifdef OSG_GL_FIXED_FUNCTION_AVAILABLE branches removed
  All #ifdef OSG_GL_MATRICES_AVAILABLE branches removed
  All #ifdef OSG_GL_VERTEX_ARRAY_FUNCS_AVAILABLE branches removed
  All GL1/GL2/GLES1/GLES2/GLES3 feature guards removed
  Only GL3 code paths remain — unconditional
